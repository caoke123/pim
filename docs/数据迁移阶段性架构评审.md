# 二、当前核心问题（必须解决）

## 2.1 数据结构双轨制（最大问题）

### ❌ 当前状态

| 系统      | Schema        |
| ------- | ------------- |
| PIM 新模块 | Shared Schema |
| 旧 API   | Legacy Schema |
| 同一业务实体  | 两套结构          |

### 结果：

* ❌ 产品数据“双人格”
* ❌ SKU 两套 ID 系统（UUID vs spu_code）
* ❌ 无法确定“真数据源”
* ❌ SyncService 写错库

---

## 2.2 数据来源混乱

### 已确认真实情况：

| 系统            | 作用             |
| ------------- | -------------- |
| Sorter        | SPU/SKU 原始数据来源 |
| PIM           | 数据加工 + 编辑      |
| Legacy Schema | 历史遗留写入目标       |
| Shared Schema | 新系统目标结构        |

### ❗ 关键问题：

> Shared products 表 **没有创建入口**

意味着：

```
Shared products ≠ 业务源头
Shared products = “被外部写入的结构”
```

---

## 2.3 SyncService 结构错误

### 当前错误逻辑：

```
R2 → Legacy products ❌（错误主链）
R2 → Shared products ❌（未建立）
```

### 真实问题：

* 同步写错目标库
* 没有统一数据管道
* 没有 Event 层

---

## 2.4 E-Catalog 数据问题

### 当前：

* 读取 Supabase（Replica）
* Supabase 数据来自未知同步
* 无法保证一致性

---

# 四、目标架构（最终统一方案）

## 4.1 架构原则（已确认）

### ✔ 单主库原则

```
Docker PostgreSQL = 唯一写入源（MASTER）
```

### ✔ 云镜像原则

```
Supabase = 只读 Replica（100%一致）
```

### ✔ 单向同步

```
Master → Replica（禁止双向）
```

---

# 五、核心架构设计（关键模块）

## 5.1 Master Data Layer（唯一真源）

### 统一表：

* `spus`
* `skus`
* `products`
* `product_skus`
* `catalogs`
* `customers`
* `distributions`
* `distribution_sku_prices`

### ❗ 关键规则：

> Shared Schema = 唯一业务模型

---

## 5.2 Legacy Schema（必须废弃）

### 包含：

* routes/products.ts
* services/sync.ts
* product_platforms
* export_records

### 状态：

```
❌ 只允许读取
❌ 禁止写入
❌ 最终删除
```

---

## 5.3 Sync Layer（重建）

### 新结构：

```
Event Publisher (PIM)
        ↓
   Sync Queue
        ↓
 Supabase Writer
```

### 特点：

* Event Driven
* 幂等写入
* 支持重试
* 支持断点恢复

---

## 5.4 Cloud Replica（Supabase）

### 职责：

* E-Catalog 查询
* BI 分析
* 灾备恢复

### 禁止：

```
❌ 写入
❌ 修改
❌ 业务逻辑
```

# 八、风险总结

## 🔴 高风险（当前系统问题）

* SyncService 写错库
* Legacy + Shared 双轨运行
* Supabase 数据来源不明确
* 没有统一写入入口

---

## 🟠 中风险

* 字段不兼容（varchar/text）
* SKU 双体系
* 外键不一致

---

## 🟢 低风险

* E-Catalog 只读
* 分拣系统稳定

# 十、结论（一句话版本）

> 当前系统最大问题不是功能，而是**数据源不统一 + 双 Schema 并存 + 同步链路缺失**。
> 本方案的核心是：

```
统一 Shared Schema + 单主库 + 单向云镜像同步
```
