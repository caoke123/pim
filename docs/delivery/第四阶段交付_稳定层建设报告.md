# 2. 交付物详情

## 2.1 全局错误体系

### 错误类层级

```
Error
 └─ AppError (基础, status + code + details + toJSON)
     ├─ BusinessError (400, 业务规则违反)
     ├─ ValidationError (400, 参数校验失败)
     ├─ NotFoundError (404, 资源不存在)
     ├─ UnauthorizedError (401)
     └─ ForbiddenError (403)
```

### ErrorCode 枚举 (15 个错误码)

| 分类 | 错误码 |
|------|--------|
| 通用 | `INTERNAL`, `NOT_FOUND`, `VALIDATION`, `UNAUTHORIZED`, `FORBIDDEN`, `CONFLICT` |
| 产品 | `PRODUCT_NOT_FOUND`, `SKU_NOT_FOUND`, `SKU_NOT_BELONG`, `INVALID_STATUS`, `INVALID_PLATFORM` |
| 数据 | `DUPLICATE_SPU_CODE`, `DUPLICATE_SKU_CODE` |
| 系统 | `DB_ERROR`, `R2_ERROR`, `SYNC_ERROR`, `EXPORT_ERROR` |

### 全局错误中间件

- `AppError` 系列 → 返回结构化 JSON: `{ success: false, error: { code, message, details } }`
- 未知错误 → 500: `{ code: "INTERNAL_ERROR", message: "服务器内部错误" }`
- 始终包含 `requestId` 和 `timestamp`

## 2.2 Zod 参数校验全覆盖

### 4 组 Schema

| Schema | 校验内容 | 规则数 |
|--------|---------|--------|
| `productListQuerySchema` | query: page, pageSize, keyword, status, platform, sortBy, order | 7 条规则 |
| `productIdParamSchema` | params: id (UUID) | 1 条规则 |
| `productBasicUpdateSchema` | body: 9 个可选字段 + refine(至少一个字段) | 10 条规则 |
| `skusUpdateSchema` | body: skus[] (1-50, skuId UUID, 7 个可选字段) | 嵌套校验 |

### 校验覆盖示例

| 场景 | 校验结果 |
|------|---------|
| `?status=invalid_status` | 400 VALIDATION_ERROR |
| `/:id` = `not-a-uuid` | 400 VALIDATION_ERROR |
| `PATCH body = {}` | 400 (至少一个字段) |
| `skus: []` | 400 (至少一个 SKU) |
| `skus.length = 51` | 400 (最多 50) |
| `sellingPrice = -1` | 400 (不能为负) |

## 2.3 Logger 系统 (Pino)

```typescript
// 应用级日志
logger.info({ port }, '服务启动')
logger.error({ err }, '错误详情')

// 请求级日志 (含 requestId)
const log = createRequestLogger(requestId)
log.info({ productId }, '查询产品详情')
```

开发环境: pino-pretty 彩色输出  
生产环境: JSON 结构化日志  
级别: DEBUG(dev) / INFO(prod)

## 2.5 requestId 全链路贯穿

```
Request → requestIdMiddleware (c.set('requestId', uuid))
  → Route (c.get('requestId') → createRequestLogger)
    → Service (logger 自动携带 requestId)
      → OperationLog
  → Error Handler (从 Context 提取 requestId)
  → Response (c.get('requestId'))
```

每个请求的响应头: `X-Request-Id: <uuid>`
